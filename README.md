# 🎙️ A simple ML Lab Voice Agent

A real-time voice AI agent built with LiveKit, designed to be a sharp, opinionated lab partner for thinking through ML and systems problems — not a polite assistant.

---

## What it does

Talk to it. It talks back. It's fast, slightly sarcastic, and will call out bad ideas.

- **Searches arXiv** for ML/AI papers on demand
- **Searches the web** via Tavily for docs, benchmarks, and current info
- **Looks up HuggingFace models** for stats, downloads, and tags
- **Does math** — latency estimates, parameter counts, cost calculations
- Keeps responses under 2–3 sentences. No lectures.

---

## Stack

| Layer | Tech |
|---|---|
| Framework | LiveKit Agents 1.4.6 |
| STT | Deepgram Nova-3 |
| LLM | Mistral Small 3.2 24B via OpenRouter |
| TTS | Deepgram Aura-2 Thalia |
| VAD | Silero |
| Turn detection | LiveKit Multilingual Model |

| Metric | Value |
|---|---|
| Avg end-to-end latency | ~3.2s |
| Avg LLM TTFT | ~1.0s |
| Avg EOU delay | ~874ms |
| Avg TTS TTFB | ~1.0s |
| Prompt tokens / session | ~14k (trimmed) |

> Tool calls (web search, arXiv) add ~1–2s on top of baseline latency.

## Setup

**1. Clone and install**
```bash
git clone https://github.com/yourname/ml-voice-agent
cd ml-voice-agent
uv sync
```

**2. Set environment variables**
```bash
cp .env.example .env
```

```env
LIVEKIT_URL=wss://your-project.livekit.cloud
LIVEKIT_API_KEY=your_key
LIVEKIT_API_SECRET=your_secret

DEEPGRAM_API_KEY=your_key
OPENROUTER_API_KEY=your_key
TAVILY_API_KEY=your_key
HF_TOKEN=your_token          # optional, for private model lookups
```

**3. Run locally**
```bash
uv run agent.py console      # console mode (no room needed)
uv run agent.py dev          # connect to LiveKit cloud
```

---

## Deploy to LiveKit Cloud

```bash
lk agent create
```

That's it. The CLI detects Python/uv, builds a Docker image, and deploys automatically.

## Optimizations applied

- **History trimming** via `llm_node` override — caps context to last 10 non-system messages
- **Tool result stripping** from history — prevents verbose search results ballooning the prompt
- **EOU tuning** — `min_endpointing_delay=0.2` reduces turn detection lag
- **Noise cancellation** — LiveKit BVC applied at room input

---

## Known limitations

- LLM latency is variable (OpenRouter routing overhead) — switching to Groq would cut TTFT by ~70%
- TTS occasionally spikes on cold WebSocket reconnections (Deepgram)
- No persistent memory across sessions
